Git

GIT BASH

Используя Git Bash, нужно перейти в корневую директорию проекта, используя команду cd "путь к проекту" , pwd для проверки папки, в которой я сейчас нахожусь.

Далее нужно прописать команду git init - эта команда создаст .git репозиторий в проекте.

git config --global user.name "Ваше Имя", git config --global user.email "ВашаПочта@mail.com"- эти команды определят информацию, которая будет использоваться при каждом commit(фиксирование изменений). Их стоит выполнить всего один раз при первичной установке Git.

git add . - для добавления всех файлов проекта в репозиторий, так же нужно создать файл в папке с проектком, под названием - git ignore в этом файле, используя текстовый редактор, можно будет прописать все файлы, которые не должны добавляться в репозиторий.

git commit - команда для фиксирования изменений в проекте. git checkout b название ветки - команда для создания новой ветки и быстрого перехода на эту ветку.
git checkout название ветки - команда для перехода на ту или иную ветку
git merge название ветки - команда, которая соеденит выбранную ветку с master веткой.
git remote add origin git@github.com:agentj29/first_git.git - для подключения удаленного репозитория.
git push origin master - для пуша проекта на удаленный репозиторий. git clone git@github.com:name/name.git - команда для загрузки проекта из удаленного репозитория

git fetch - для притягивания веток проекта.
git pull - команда для выгрузки комитов из удаленного репозитория в локальный проект. ( состоит из fetch + merge)
